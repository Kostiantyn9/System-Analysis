# Розділ 8. Обґрунтування вибору типу бази даних

## 8.1 Вибір моделі збереження даних

Для інформаційної системи інтернет-магазину обрано **реляційну модель збереження даних (SQL)**.

Обґрунтування вибору:

1. Дані мають чітку структуру та зв’язки:
   - користувачі
   - товари
   - замовлення
   - кошики
   - категорії
   - бренди

2. Необхідна підтримка зв’язків між таблицями (JOIN):
   - User -> Order
   - Order -> OrderItem
   - Product -> Category

3. Важлива узгодженість даних (ACID-властивості):
   - гарантія збереження транзакцій
   - коректна обробка платежів
   - запобігання втраті замовлень

4. Структура даних не є надто гнучкою або неструктурованою, тому використання NoSQL не є критично необхідним.

---

## 8.2 Вибір конкретної СУБД

Для реалізації системи доцільно використати **PostgreSQL**.

Причини вибору:

- підтримка складних SQL-запитів
- надійна реалізація транзакцій (ACID)
- підтримка індексування для швидкого пошуку
- можливість масштабування
- відкритий код та широка підтримка

---

## 8.3 Можливість використання додаткових технологій

Для підвищення продуктивності система може додатково використовувати:

- **Redis** - для кешування популярних товарів або сесій користувачів
- Реплікацію PostgreSQL - для підвищення відмовостійкості

Таким чином, використання SQL-бази даних (PostgreSQL) є оптимальним рішенням для інтернет-магазину з точки зору структури даних, узгодженості та масштабованості.
